# PDF Multimodal Artifact Store Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend PDF artifact persistence so the pipeline can preserve raw images, chart candidates, and table assets for later post-processing without changing the pure `load_pdf_chunks(path: str) -> list[Chunk]` contract.

**Architecture:** Keep the shared `Chunk` model text-first and lightweight. Add a richer explicit helper in the PDF ingestion module that uses Docling's document object to save text artifacts plus `elements.jsonl` and asset files under ignored `.data`; chunks reference assets through metadata IDs only. V1 stores raw assets and table text/CSV; it does not enable heavy picture-captioning or chart-extraction models.

**Tech Stack:** Python 3.12, uv, Pydantic v2, Docling, Pillow via Docling runtime, Ruff, mypy, pytest.

## Chunk Metadata Contract

Only the new multimodal helper adds asset reference metadata. Existing `load_pdf_chunks()` and `save_pdf_ingestion_artifacts()` behavior stays unchanged.

For chunks generated by `save_pdf_multimodal_artifacts()`, metadata may include:

```python
{
    "asset_ids": ["pdf_example_table_0001"],
    "has_image": False,
    "has_table": True,
    "has_chart": False,
}
```

Rules:

- Never store binary image data in `Chunk`.
- Never store a DataFrame/table object in `Chunk`.
- `asset_ids` is a list of `PdfElementArtifact.element_id` values.
- If no asset maps cleanly to a chunk, keep `asset_ids=[]` and all flags `False`.
- Existing shared metadata keys such as `source`, `source_type`, `file_name`, `page`, `section`, `parser`, and `chunk_index` must remain present.

- [ ] **Step 3: Implement Docling multimodal parsing**

Add to `DoclingMarkdownParser`:

```python
def parse_to_document(self, path: Path) -> ParsedPdfDocument:
    try:
        converter_class = _get_document_converter()
        converter = converter_class(
            format_options={
                InputFormat.PDF: PdfFormatOption(
                    pipeline_options=PdfPipelineOptions(
                        generate_page_images=True,
                        generate_picture_images=True,
                    )
                )
            }
        )
        result = converter.convert(path)
        markdown = cast(str, result.document.export_to_markdown())
        return ParsedPdfDocument(markdown=markdown, document=result.document)
    except Exception as exc:
        raise RuntimeError(f"Failed to parse PDF with Docling: {path}") from exc
```

Imports needed:

```python

from docling.document_converter import PdfFormatOption
```

Keep lazy import behavior if import cost is a concern: import these inside the method.

- [ ] **Step 2: Add public helper**

Add:

```python
def save_pdf_multimodal_artifacts(
    path: str,
    *,
    output_root: str | Path | None = None,
    run_id: str | None = None,
) -> PdfMultimodalArtifactManifest:
    pdf_path = Path(path)
    _validate_pdf_path(pdf_path)
    parsed = DoclingMarkdownParser().parse_to_document(pdf_path)
    return _save_pdf_multimodal_artifacts_from_document(
        pdf_path,
        markdown=parsed.markdown,
        doc=parsed.document,
        output_root=output_root,
        run_id=run_id,
    )
```

- [ ] **Step 3: Add internal writer**

Implement `_save_pdf_multimodal_artifacts_from_document(...)`:

- Validate/create run directory with the same safe stem/run-id rules as current helper.
- Write `parsed.md`.
- Extract `chunks = _chunks_from_markdown(path, markdown)`.
- Extract elements from `doc.iterate_items(traverse_pictures=True)`:
  - Label `table` -> kind `table`.
  - Label `chart` -> kind `chart`.
  - Label `picture` -> kind `image`.
  - Ignore other labels.
- Save assets:
  - Image/chart: `get_image(doc)` to PNG if not `None`.
  - Table: `export_to_markdown(doc)` to `.md`, `export_to_dataframe(doc).to_csv(..., index=False)` to `.csv`, and `get_image(doc)` to `.png` if available.
- Write one `PdfElementArtifact.model_dump(mode="json")` per line to `elements.jsonl`.
- Enrich chunks with metadata keys `asset_ids`, `has_image`, `has_table`, `has_chart`.
- Write enriched chunks to `chunks.jsonl`.
- Write `PdfMultimodalArtifactManifest` to `manifest.json`.

- [ ] **Step 4: Keep mapping simple for v1**

Use a deterministic v1 mapping:

- If there are chunks, attach all extracted `element_id`s to the first chunk.
- Set flags based on extracted element kinds.
- Leave each `PdfElementArtifact.chunk_ids` as `[first_chunk.chunk_id]`.
- If there are no chunks, write elements with `chunk_ids=[]`.

This is intentionally simple and testable. Page-aware chunk alignment can be optimized later because current baseline chunks have `page=None`.

## Acceptance Criteria

- `load_pdf_chunks(path: str) -> list[Chunk]` remains unchanged and side-effect free.
- `save_pdf_ingestion_artifacts()` still produces text-only `parsed.md`, `chunks.jsonl`, and `manifest.json`.
- `save_pdf_multimodal_artifacts()` produces `parsed.md`, `chunks.jsonl`, `manifest.json`, `elements.jsonl`, and `assets/`.
- `elements.jsonl` validates with `PdfElementArtifact`.
- `manifest.json` validates with `PdfMultimodalArtifactManifest`.
- `chunks.jsonl` validates with shared `Chunk`; asset data is referenced only through metadata.
- Tests use fake document/items and do not require real Docling conversion.
- Vietnamese PDF README documents the new workflow.
