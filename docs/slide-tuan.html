Tổng quan

Vòng đời metadata: sinh → lưu → áp dụng → đo

Offline · Ingest

Parse (PDF/HTML → text + chunk) → Rule base extract metadata (cấu trúc/URL/regex) → LLM Extract (entities, questions, keywords, document_type) → Store (Neon + Qdrant + S3).

Online · Trả lời

Query → detect entity → hard filter + hybrid (dense + sparse) + question-index → fusion RRF → boost → rerank → trả lời có trích dẫn.

Sản phẩm

Phân loại trang

Store: 3 backend + hybrid

Neon / Postgres

Source of truth. Bảng documents + chunks (text + metadata JSONB, GIN + full-text).

Qdrant

Vector: dense + sparse mỗi chunk + payload + keyword index (entities_canonical, dedup layer).

S3

Raw & backup: raw/*.pdf · parsed/*.md · chunks.jsonl · manifest.json.

Phương pháp · lưu vector

upsert_dense_embeddings: cách ghi Qdrant

# mỗi chunk → 1 point (theo batch DENSE_EMBED_BATCH_SIZE) id = UUID5(storage_chunk_id) # tất định, ổn định khi chạy lại dense = embed(chunk.text) # hoặc vector tính sẵn sparse = _sparse_vector(_bm25_index_text(chunk)) payload = { text, document_id, metadata{}, _embedding_profile }
  • Sparse = bag-of-words băm: Counter(tokens) → index = sha256(token)[:8] % 2^31-1, value = tần suất.
  • _embedding_profile (provider, model, dimensions) lưu kèm để validate lúc query, chặn lệch model/chiều vector.
  • Id tất định nên re-upsert ghi đè đúng chỗ, không nhân bản.

Phương pháp · boosting

Công thức 2 hệ số

score' = norm(score) × clamp( fdoctype × fdedup , 0.7, 1.4 )
  • fdoctype: ma trận query_type → document_type (query "faq" ưu chunk faq ×1.3, manual ×1.1, lệch loại ×0.85); query "unknown" dùng hệ số mặc định.
  • fdedup: chunk duplicate_candidate ×0.8 (đẩy bản trùng xuống).
Áp dụng metadata · questions · Huy

Question-index: đường thứ 3

Khớp câu hỏi người dùng với câu hỏi của chunk (question ↔ question), fuse RRF.

Collection phụ Qdrant (ưu tiên)

{collection}_questions: 1 point / câu hỏi, dense-embed, payload denormalize chunk cha. Bền, gần như 0 RAM, embed 1 lần. build_question_index.py5.803 câu

In-memory fallback

Chưa build thì scroll + embed toàn collection lần đầu rồi cache. Trace source=qdrant_native|in_memory.

Phương pháp · question-index

Cách hoạt động

# build mỗi chunk → ≤ 4 questions → embed → point{ parent_chunk_id, text, q } # query q_vec = reuse dense vector của hybrid # embed 1 lần dùng cho cả 2 hits = search(questions_collection, q_vec) gom theo parent → max-score/chunk → cắt QUESTION_MIN_SCORE results(retriever="question") → rrf_fusion_nway([hybrid, question])
  • Dedup max-score: nhiều câu hỏi trỏ cùng 1 chunk thì giữ điểm cao nhất.
  • RRF: hợp nhất theo thứ hạng 1 / (k + rank), k = 60, nên các thang điểm khác nhau vẫn fuse công bằng.
  • Tối ưu: query chỉ embed 1 lần, dùng chung cho hybrid và question-index.
Áp dụng metadata · keywords · BM25

Bơm keywords vào sparse

  • Ý tưởng: nối keywords (LLM) vào cuối text mà sparse index bao phủ, tăng tín hiệu lexical; không đụng dense, không đụng text gốc.
  • Với Qdrant là cờ lúc INDEX: sparse vector lưu sẵn nên đổi RETRIEVAL_BM25_AUGMENT_KEYWORDS phải chạy reupsert_sparse.py (update_vectors sparse-only, không re-embed dense).
